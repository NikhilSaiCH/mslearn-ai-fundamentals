## Task 1: Cleaning the Time Series & Classification Dataset

In this task,

1. Open the link below in your browser to access **Azure Machine Learning Studio**:

    ```
    https://ml.azure.com/
    ```

1. If prompted to sign in, enter your credentials.
 
   - **Email/Username:** <inject key="AzureAdUserEmail"></inject>
 
     ![Enter Your Username](../images/lesson10-e1t1p1(1).png)
 
   - **Password:** <inject key="AzureAdUserPassword"></inject>
 
     ![Enter Your Password](../images/lesson10-e1t1p1(2).png)
 
1. If prompted to stay signed in, you can click **No**.

   ![](../images/lesson10-e1t1p1(3).png)

1. From the left navigation pane, select **Workspaces (1)** and then click on workspace **SmartCities (2)**.

     ![](../images/L13-01.png)

1. From the left navigation pane, go to **Assets**, select **Data (1)**, then open the **Datastores (2)** tab and click **+ Create (3)**.

     ![](../images/L13-01-1.png)

1. On the **Create datasore** pane, enter the following details and then click **Create (9)**:

    - Datastore name: **traffic_sc_dataset (1)**
    - Datastore type: **Azure SQL database (2)**
    - Account selection method: **From Azure subscription (3)**
    - Subscription ID: Select the default subscription **(4)** 
    - Server name / database name: Select **smartcity-sqlserver-<inject key="Deployment ID" enableCopy="false" /> (5)**
    - Authentication type: **SQL Authentication (6)**
    - User ID: **<inject key="SQL Admin Login" enableCopy="false" /> (7)**
    - Password: **<inject key="SQL Admin Password" enableCopy="false" /> (8)**

        ![](../images/L13-01-2.png)

1. Once the datastore is created successfully, the new datastore will appear in the **Datastores** tab.

    ![](../images/lesson10-e1t1p5.png)

1. Select **Designer (1)** from the left navigation menu, then under the **New pipeline** section (Classic prebuilt), click **+ Create a new pipeline using classic prebuilt components (2)**.

    ![](../images/L13-02.png)

1. Under the **Data (1)** tab, click on **+ (2)** icon.

    ![](../images/L13-03.png)

1. Provide data asset name as **Traffic_TimeSeries_SC_with_Classification (1)** and click on **Next (2)**.

    ![](../images/L13-04.png)

1. Choose **From local files (1)** option for data source and click on **Next (2)**.

    ![](../images/L13-05.png)

1. Select **workspaceblobstore (1)** for data store and click on **Next (2)**.

    ![](../images/L13-06.png)

1. Next, click on **Upload files or folder (1)** and select **Upload files (2)**.

    ![](../images/L13-07.png)

1. Navigate to **C:\AllFiles\Unit 1-Lesson 13 - Cleaning Traffic Datasets in Azure ML Designer (1)** folder, select **Traffic_TimeSeries_SC_with_Classification (2)** CSV file and click on **Open (3)**.

    ![](../images/L13-08.png)

1. After uploading file, make sure that **Traffic_TimeSeries_SC_with_Classification.csv (1)** file is visible under upload list. Then click on **Next (2)**.

    ![](../images/L13-09.png)

1. Check the preview of Traffic_TimeSeries_SC_with_Classification.csv file and click on **Next**.

1. Review the schema of Traffic_TimeSeries_SC_with_Classification.csv file and click on **Next**.

    ![](../images/L13-10.png)

1. Review the settings done till now for data asset and click on **Create**.

    ![](../images/L13-11.png)

1. Under the **Data** tab, find your dataset **Traffic_TimeSeries_SC_with_Classification** and drag it onto the pipeline canvas to use it as the starting point of your machine learning workflow.

    ![](../images/L13-12.png)
